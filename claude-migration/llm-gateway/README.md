# Claude Code local LLM gateway

Adds DeepSeek V4, free OpenRouter models, and paid models to the Claude Code `/model` picker while keeping Opus/Sonnet/Haiku working natively.

## Architecture

```
Claude Code  ->  router (127.0.0.1:4000)
    GET /v1/models      -> static Anthropic-format list of all models
    POST /v1/messages   model "claude-deepseek-*" -> RAW passthrough to api.deepseek.com/anthropic
                        model "claude-free-*"     -> LiteLLM -> OpenRouter
                        otherwise (Opus/Sonnet/Haiku) -> RAW passthrough to api.anthropic.com
LiteLLM (internal :4000) -> OpenRouter (only free/paid models)
```

**DeepSeek V4 goes directly to DeepSeek's native Anthropic endpoint**
(`https://api.deepseek.com/anthropic`) with `x-api-key` -- byte-for-byte passthrough,
just like Anthropic. The router rewrites the model id (`claude-deepseek-v4-pro` ->
`deepseek-v4-pro`) so DeepSeek does NOT silently remap Pro to Flash. This is the
setup DeepSeek officially recommends for Claude Code (thinking, tool use, streaming preserved).

Anthropic traffic is also byte-for-byte passthrough. Only free/paid models use
LiteLLM (Anthropic <-> OpenAI translation for OpenRouter).

## Files

- `docker-compose.yml`      two services: router (built) + litellm
- `router/app.py`           the routing proxy with model lists (DEEPSEEK_MODELS, FREE_MODELS, PAID_MODELS)
- `litellm-config.yaml`     11 OpenRouter models (10 free + 1 paid). DeepSeek is NOT here -- it's native passthrough in the router.
- `.env`                    API keys (chmod 600, auto-generated by install.sh)
- `.env.template`           template for manual setup

## Quick Start

Run the installer from the parent directory:

```bash
./start.sh
```

Or from inside the extracted folder:

```bash
./claude-migration/start.sh
```

The installer (`install.sh`) handles everything:
1. Node.js + npm installation
2. Claude Code installation
3. Docker checks
4. Interactive API key input (Anthropic, OpenRouter, DeepSeek)
5. Docker gateway startup
6. Gateway healthcheck + model verification
7. Configuration copy to `~/.claude/`
8. **Environment variables in `~/.bashrc`** (ANTHROPIC_BASE_URL, etc.)

After installation, restart your terminal or run `source ~/.bashrc`.

## Claude Code wiring (automatic)

The installer sets these in `~/.bashrc`:

```
ANTHROPIC_BASE_URL=http://127.0.0.1:4000
CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1
ANTHROPIC_AUTH_TOKEN=<gateway-token>
```

And adds a `claude-direct` shell function to bypass the gateway.

## Manage

```bash
cd llm-gateway

docker compose ps                    # status
docker compose logs -f router        # routing decisions (deepseek vs litellm vs anthropic)
docker compose restart               # restart
docker compose down                  # stop (disables all models until back up!)
docker compose up -d --build         # start / rebuild
```

## Add / change models

**Free / paid (OpenRouter, via LiteLLM):**
1. Add to `litellm-config.yaml` (model_name must start with `claude-`)
2. Add to `FREE_MODELS` or `PAID_MODELS` in `router/app.py`
3. Rebuild: `docker compose up -d --build`

**DeepSeek (native passthrough, NOT via LiteLLM):**
1. Add to `DEEPSEEK_MODELS` in `router/app.py`. The Claude id must be
   `claude-` + the real DeepSeek model id (router strips `claude-` before
   forwarding, e.g. `claude-deepseek-v4-pro` -> `deepseek-v4-pro`).
2. Rebuild: `docker compose up -d --build router`

## Troubleshooting

### Gateway not running
```bash
cd llm-gateway && docker compose logs
```

### Free models not working
- Verify OpenRouter key: https://openrouter.ai/dashboard
- Logs: `docker compose logs -f router`

### DeepSeek not working
- Was DeepSeek API key provided during install?
- Check `llm-gateway/.env`: `DEEPSEEK_API_KEY=...`
- **DeepSeek goes directly to api.deepseek.com/anthropic** -- verify your DeepSeek account has credits
- DeepSeek is native passthrough in the router (NOT litellm). Logs: `docker compose logs -f router` (look for `-> deepseek`)

### Claude models not working
- Gateway must be running (even for Anthropic models!)
- `docker ps | grep llm-gateway`

### Environment variables not working
- Run: `source ~/.bashrc`
- Verify: `echo $ANTHROPIC_BASE_URL` should show `http://127.0.0.1:4000`
